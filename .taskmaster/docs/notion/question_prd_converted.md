# 4. Requirements

## 4.1 기능 요구사항 (Must)

✅ **확정:** 다음 기능이 필수입니다:

- **FR-01:** 웹툰 프로젝트 생성 및 관리
  - 수용 기준: 프로젝트명, 장르, 타겟 플랫폼, 마감일 입력 가능

- **FR-02:** 에피소드 단위 원고 입력/업로드
  - 수용 기준: 텍스트 입력과 파일 업로드 모두 지원, 에피소드 단위 분리 저장

- **FR-03:** 원고 버전 관리
  - 수용 기준: 에피소드별 버전 히스토리 조회 및 복원 가능

- **FR-04:** AI 기반 콘텐츠 생성
  - 수용 기준: 이미지, 텍스트, 메타데이터 자동 생성

- **FR-05:** 플랫폼별 업로드/배포 연동
  - 수용 기준: 네이버웹툰, 카카오페이지 등 주요 플랫폼 API 연동

# 5. Acceptance Criteria

## 5.1 핵심 수용 기준

✅ **확정:** 다음 기준을 만족해야 합니다:

1. **프로젝트 생성:** 작품명, 장르, 타겟 플랫폼, 마감일을 입력받아 프로젝트 생성 가능
2. **원고 입력:** 텍스트 입력과 파일 업로드를 모두 지원하고 에피소드 단위로 분리 저장
3. **AI 생성:** 프롬프트 기반으로 이미지, 텍스트, 메타데이터 생성 가능
4. **플랫폼 연동:** 최소 1개 이상의 주요 웹툰 플랫폼 API 연동 및 업로드 가능
5. **버전 관리:** 에피소드별 버전 히스토리 조회 및 이전 버전 복원 가능
