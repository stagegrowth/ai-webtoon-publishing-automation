# As-Is 요약 (고객 확인용)

이 문서는 현재 수동 발행 흐름을 정리한 내용입니다. 확인 후 결정이 필요한 항목에 체크·의견을 주시면 다음 단계(자동화 범위·우선순위) 설계에 반영하겠습니다.

## Current Manual Flow

현재 웹툰 발행 시 담당자가 수행하는 주요 단계는 아래와 같습니다(ASIS-01 ~ ASIS-12 기준).

1. **로그인** 후 관리 콘솔에 접속합니다.
2. **워크스페이스·프로젝트**를 선택합니다.
3. **작품(Work)**을 새로 만들거나 기존 작품을 선택합니다.
4. **에피소드 메타**(제목, 요약 등)를 입력하고, **원고/이미지 파일**을 업로드합니다.
5. **원고 검증**을 실행해 필수 항목·분량·규칙 위반 여부를 확인합니다.
6. (선택) **콘티/구조**를 넣거나 수정합니다.
7. **에셋(이미지 등)**을 추가·첨부하고, **최종 검토·수정**을 합니다.
8. **발행할 플랫폼·채널**을 선택하고 설정합니다.
9. **발행(업로드) 요청**을 실행하면 외부 플랫폼으로 전송됩니다.
10. **발행 결과**를 조회해 성공·실패·링크를 확인합니다.
11. **완료 후** 플랫폼 노출·링크 등을 수동으로 한 번 더 확인합니다.

위 단계는 모두 웹 관리 화면에서 수동으로 진행되는 흐름입니다.

## Top Exceptions / Rework

자주 발생하는 예외와 현재 대응, 자동화 시 기대 동작, 고객 결정이 필요한 사항을 정리했습니다.

| Exception | When it happens (Step) | Current handling | Desired automation behavior | Customer decision needed |
|-----------|-------------------------|------------------|-----------------------------|---------------------------|
| **EX-01 Validation** | 메타 입력·검증 (ASIS-04a, ASIS-05) | 화면에 오류 표시 후 수동 수정 | 실패 시 즉시 피드백·로그 저장, 재입력 유도 | 검증 실패 시 알림 채널(이메일/슬랙 등) 선호 |
| **EX-02 Upload** | 파일·에셋 업로드 (ASIS-04b, ASIS-07) | 실패 안내 후 재업로드 | 자동 재시도·실패 시 큐 보관 후 알림 | 재시도 횟수·타임아웃 기준 허용 범위 |
| **EX-03 Duplicate** | 에피소드 등록·발행 요청 (ASIS-04a, ASIS-10) | 중복 안내 후 사용자 선택 | 중복 시 스킵 / 덮어쓰기 / 사용자 확인 중 선택 | 중복 시 정책: 스킵 / 덮어쓰기 / 반드시 확인 중 어떤 방식? |
| **EX-04 Permission** | 로그인 이후 전 단계 | 권한 안내 후 관리자 요청 | 서비스 계정으로 자동 수행, 실패 시 담당자 알림 | 자동화용 계정·역할 사용에 대한 정책 (허용 여부) |
| **EX-05 Downstream** | 발행·외부 연동 (ASIS-10, ASIS-06/07) | 오류·재시도 안내 후 수동 대응 | 재시도·일부 실패 시 알림, 필요 시 수동 개입 | 부분 실패 시: 전부 롤백 / 성공분만 반영 / 수동 확인 후 결정 중 선호 |

## Automation Boundary

어떤 단계를 **완전 자동**으로 할지, **반자동**(사람 확인 포함)으로 할지, **자동화 제외**로 둘지 구분해 두었습니다. 고객 확인 후 최종 확정할 예정입니다.

| 구분 | 후보 단계 (Step ID) | 설명 |
|------|---------------------|------|
| **완전 자동** | ASIS-02(기본값 고정), ASIS-04a/04b(입력 패키지 기반), ASIS-05(검증), ASIS-10(발행 요청), ASIS-11(결과 조회) | 트리거(예: 폴더 드롭·스케줄)만 주면 입력 패키지 기준으로 끝까지 수행 |
| **반자동** | ASIS-03(작품 생성/선택), ASIS-08(최종 검토), ASIS-09(플랫폼 선택) | 자동 제안 또는 기본값 적용 후, **사람이 확인·승인**하는 단계로 두는 후보 |
| **제외** | ASIS-01(로그인 → API 키/서비스 계정으로 대체), ASIS-06(콘티·선택 단계), ASIS-12(Post-Check) | 초기 자동화 범위에서 빼거나, 별도 정책(예: 로그인은 계정 관리로 이전)으로 처리하는 후보 |

### Open Questions (고객 결정 필요)

1. **자동화 시작 시점**: 발행을 "폴더/파일이 준비되면 자동 실행"으로 할지, "담당자가 버튼/스케줄 한 번 실행"으로 할지 선호가 있나요?
2. **반드시 사람 확인이 필요한 단계**: 최종 검토(ASIS-08), 플랫폼 선택(ASIS-09) 중 자동화 후에도 **반드시** 사람이 확인해야 하는 단계가 있나요?
3. **예외 발생 시 알림**: 검증 실패·업로드 실패·권한 오류 시 알림을 받을 채널(이메일, 슬랙, 내부 시스템 등)과 담당자 지정 선호가 있나요?
4. **중복 발행 정책**: 같은 에피소드를 다시 발행 요청했을 때 "스킵 / 덮어쓰기 / 항상 사용자 확인" 중 어떤 정책을 원하시나요?
5. **부분 실패 시 처리**: 일부 플랫폼만 실패했을 때 "전부 롤백 / 성공한 것만 반영 / 수동 확인 후 결정" 중 선호하는 방식을 알려 주세요.
