# System Feature List (시스템 기능 목록)

## 1. 문서 목적
웹툰 등록 자동화 **시스템이 제공하는 기능**을 도메인별로 나열합니다. 시스템 컨텍스트와 맞춰 범위·우선순위 논의와 설계에 활용합니다. (기능 요구사항 FR은 별도 페이지에서 도출·정리 예정.)

---

## 2. 기능 요약

| 영역 | 기능 요약 |
|------|-----------|
| 입력·소스 | 원고·메타 소스 지정, 메타·파일 읽기, 실행 단위(작품/회차) 선택 |
| 사전 검증 | 업로드 전 누락·형식·파일 규격 검증, 검증 결과 조회 |
| 등록 실행 | 플랫폼별 로그인·메타 입력·파일 업로드·제출 자동 실행, 재시도 |
| 결과·리포트 | 제출/완료 상태 저장, 실패 사유·재처리 구분 리포트, 알림 |
| 설정·운영 | 플랫폼별 설정·검증 규칙·재시도 정책, 실행 이력·모니터링 |

---

## 3. 기능 목록 (상세)

### 3.1 입력·소스 (Input & Source)

| ID | 기능 | 설명 | 비고 |
|----|------|------|------|
| F-INPUT-01 | 원고·메타 소스 지정 | 로컬/공유 경로·엑셀 등 메타 소스 위치를 설정하거나 실행 시 지정 | 설정 또는 실행 파라미터 |
| F-INPUT-02 | 메타·파일 읽기 | 지정된 소스에서 메타데이터(제목·작가·카테고리 등)와 원고 파일 목록을 읽음 | 폴더 구조·엑셀 스키마에 의존 |
| F-INPUT-03 | 실행 단위 선택 | 작품·회차·플랫폼 단위로 실행 대상을 선택(전체/일부/단일) | 배치·단건 실행 지원 |

---

### 3.2 사전 검증 (Pre-validation)

| ID | 기능 | 설명 | 비고 |
|----|------|------|------|
| F-VALID-01 | 업로드 전 사전 검증 | 메타 누락·형식 오류·파일 규격(크기·확장자 등)을 업로드 전에 검사 | — |
| F-VALID-02 | 검증 규칙 설정 | 플랫폼별·공통 검증 규칙(필수 필드·금칙어·파일 규격)을 설정으로 관리 | 변경 가능 항목 |
| F-VALID-03 | 검증 결과 조회 | 건별·요약 검증 결과를 화면·리포트로 확인 | — |

---

### 3.3 등록 실행 (Registration Execution)

| ID | 기능 | 설명 | 비고 |
|----|------|------|------|
| F-EXEC-01 | 플랫폼별 등록 자동 실행 | 로그인·워크스페이스/작품 선택·메타 입력·파일 업로드·제출을 플랫폼별로 자동 수행 | 핵심 기능 |
| F-EXEC-02 | 재시도 | 일시적 오류(네트워크·타임아웃 등)에 대해 설정된 횟수·간격으로 재시도 | 안정성 |
| F-EXEC-03 | 수동 전환 구분 | 재시도 가능 오류와 수동 처리 필요 오류를 구분해 상태·리포트에 반영 | — |
| F-EXEC-04 | 제출/완료 처리 및 상태 저장 | 제출 완료·실패·보류 등 상태를 저장하고 이력 조회 가능 | — |

---

### 3.4 결과·리포트 (Result & Report)

| ID | 기능 | 설명 | 비고 |
|----|------|------|------|
| F-REPORT-01 | 실패 사유 리포트 | 건별 성공/실패·실패 사유·URL·에러 메시지를 리포트로 제공 | — |
| F-REPORT-02 | 재시도 가능 vs 수동 전환 구분 | 오류 유형별로 재시도 대상·수동 전환 대상을 구분해 표시 | — |
| F-REPORT-03 | 실행 이력 조회 | 실행 일시·대상·결과 요약을 이력으로 조회 | 추적성 |
| F-REPORT-04 | 알림 | 실패·완료 등 이벤트를 이메일·슬랙 등 설정된 채널로 알림 | 운영성 |

---

### 3.5 설정·운영 (Configuration & Operations)

| ID | 기능 | 설명 | 비고 |
|----|------|------|------|
| F-CONFIG-01 | 플랫폼별 설정 | 플랫폼별 URL·계정 정보·기본값·필수 필드 매핑 등 설정 관리 | 확장성 |
| F-CONFIG-02 | 검증·재시도 정책 설정 | 검증 규칙·재시도 횟수·타임아웃·수동 전환 기준 등 정책 설정 | 변경 가능 항목 |
| F-CONFIG-03 | 실행 이력·모니터링 | 실행 이력·상태·에러 집계를 운영자가 확인할 수 있는 화면·도구 | 추적성·운영성 |
| F-CONFIG-04 | 계정·권한 관리 | 플랫폼 로그인 계정·API 키 등 자격 정보의 안전한 저장·사용 | 보안 |

---

## 4. MVP·우선순위 (참고)
범위 확정 시 아래를 기준으로 1차(MVP) 기능을 선정할 수 있습니다.
- **필수**: F-INPUT-01~02, F-VALID-01, F-EXEC-01·04, F-REPORT-01·02 (최소 1개 플랫폼)
- **조건부**: F-EXEC-02·03(재시도·구분), F-REPORT-04(알림) — 운영 정책에 따라
- **확장**: F-INPUT-03(선택 실행), F-CONFIG-01~04(설정 UI·이력) — 2차 범위
