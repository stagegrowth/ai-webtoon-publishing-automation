# Actor List (액터 목록)

## 1. 문서 목적
웹툰 등록 자동화 **시스템과 상호작용하는 액터**를 구분해 나열합니다. Use Case 도출·아키텍처 경계 설계에 사용합니다.

---

## 2. 액터 구분

| 구분 | 설명 |
|------|------|
| **Primary Actor (주 액터)** | 시스템에 요청을 시작하고, 목표를 달성하기 위해 시스템을 사용하는 역할. 사람 또는 외부 트리거. |
| **Secondary Actor (보조 액터)** | 시스템이 목표를 달성하는 과정에서 **응답을 받는** 쪽. 시스템이 요청을 보내고 응답·데이터를 받는 외부 시스템. |

---

## 3. Actor 목록

### 3.1 Primary Actor (주 액터)

| ID | 액터 | 설명 | 시스템과의 관계 |
|----|------|------|-----------------|
| ACT-01 | 고객 운영팀 | 웹툰을 다수 플랫폼에 등록하는 실무·의사결정 담당. 원고·메타를 준비하고 등록 실행을 지시하며 결과를 확인·재작업 지시한다. | 소스 지정·실행 지시·결과 확인·재작업 지시. UI·리포트·알림을 통해 시스템 사용. |
| ACT-02 | 개발/운영 담당 | 자동화 시스템을 설계·구현·운영하는 담당. 설정·배포·모니터링·장애 대응을 수행한다. | 검증 규칙·플랫폼별 설정·스케줄·배포, 실행 이력·모니터링. 설정·운영 도구를 통해 시스템 사용. |

---

### 3.2 Secondary Actor (보조 액터)

시스템이 유스케이스를 수행할 때 **요청을 보내고 응답을 받는** 외부 시스템입니다.

| ID | 액터 | 설명 | 시스템과의 관계 |
|----|------|------|-----------------|
| ACT-S01 | 원고·메타 소스 | 원고 파일·메타데이터(엑셀 등)가 있는 로컬/공유 스토리지·폴더. | 시스템이 읽기 요청(경로·파일 접근)을 하고, 메타·파일 내용을 받는다. |
| ACT-S02 | 플랫폼 포털 | 카카오·네이버·레진·교보 등 웹툰 등록 포털(웹 UI 또는 API). | 시스템이 로그인·메타 입력·업로드·제출 요청을 보내고, 성공/실패·URL·상태를 받는다. |
| ACT-S03 | 결과 리포트·알림 채널 | 실행 이력·실패 사유를 전달하는 채널(파일·이메일·슬랙 등). | 시스템이 리포트·알림을 보내고, (필요 시) 전송 결과를 받는다. |

---

## 4. 액터–시스템 경계 요약

```text
[Primary]
  ACT-01 고객 운영팀     ── 사용 ──▶  웹툰 등록 자동화 시스템
  ACT-02 개발/운영 담당  ── 사용 ──▶  웹툰 등록 자동화 시스템

[Secondary]
  웹툰 등록 자동화 시스템  ── 요청/응답 ──▶  ACT-S01 원고·메타 소스
  웹툰 등록 자동화 시스템  ── 요청/응답 ──▶  ACT-S02 플랫폼 포털
  웹툰 등록 자동화 시스템  ── 전송 ──▶  ACT-S03 결과 리포트·알림 채널
```
