# Use Case List (유스케이스 목록)

## 1. 문서 목적
웹툰 등록 자동화 **시스템에 대한 유스케이스(Use Case)**를 액터별로 나열합니다. 기능 범위·시나리오 설계와 Actor List와 함께 참고합니다.

---

## 2. Use Case 요약

| 주 액터 | Use Case 수 | 요약 |
|---------|-------------|------|
| 고객 운영팀 (ACT-01) | 8 | 소스 지정·실행 지시·검증·결과 확인·이력 조회·재작업 판단·자격 정보 변경·**요소 ID(셀렉터) 맵핑 등록·수정** |
| 개발/운영 담당 (ACT-02) | 5 | 설정·검증 규칙·배포·이력·모니터링·계정·권한 정책 관리 |

---

## 3. Use Case 목록 (상세)

### 3.1 고객 운영팀 (ACT-01) — Primary

| ID | Use Case | 설명 | 관련 보조 액터 |
|----|----------|------|----------------|
| UC-01 | 원고·메타 소스 지정 | 실행에 사용할 원고·메타 소스(경로·엑셀 등)를 지정한다. | ACT-S01 |
| UC-02 | 등록 실행 지시 | 지정된 소스·대상(작품·회차·플랫폼)으로 등록 실행을 시작한다. | ACT-S01, ACT-S02 |
| UC-03 | 사전 검증 실행 | 업로드 전에 메타·파일 규격 검증만 수행하고 결과를 확인한다. | ACT-S01 |
| UC-04 | 실행 결과·리포트 확인 | 실행 완료 후 성공/실패·실패 사유·URL·재처리 구분을 리포트로 확인한다. | ACT-S03 |
| UC-05 | 실행 이력 조회 | 과거 실행 일시·대상·결과 요약을 이력으로 조회한다. | — |
| UC-06 | 실패 건 재작업 판단 | 리포트를 보고 재시도할지·수동 처리할지 판단하고, 필요 시 재실행을 지시하거나 수동 조치한다. | — |
| UC-07 | 플랫폼 로그인 자격 정보(비밀번호 등) 변경 | 플랫폼 로그인 비밀번호·API 키 등 자격 정보 **값**을 설정 화면에서 직접 변경·입력한다. 비밀번호 변경 시 개발자 수정 없이 고객 운영팀이 대응할 수 있다. | — |
| UC-07b | 플랫폼별 요소 ID(셀렉터) 맵핑 등록·수정 | 등록 대상 플랫폼 웹의 요소 ID(또는 셀렉터)가 변경되었을 때, **변경된 요소 ID 맵핑**을 사용자 UI에서 직접 등록·수정한다. 개발자 수정 없이 고객 운영팀이 시스템을 복구·유지할 수 있다. | — |

### 3.2 개발/운영 담당 (ACT-02) — Primary

| ID | Use Case | 설명 | 관련 보조 액터 |
|----|----------|------|----------------|
| UC-08 | 플랫폼별 설정 관리 | 플랫폼별 URL·계정 정보·기본값·필수 필드 매핑 등 설정을 등록·수정한다. | — |
| UC-09 | 검증·재시도 정책 설정 | 검증 규칙(필수 필드·금칙어·파일 규격), 재시도 횟수·타임아웃·수동 전환 기준을 설정한다. | — |
| UC-10 | 배포·배치 실행 | 새 버전 배포 또는 스케줄/배치에 따른 실행 환경을 구성·실행한다. | ACT-S02, ACT-S03 |
| UC-11 | 실행 이력·모니터링 확인 | 실행 이력·상태·에러 집계를 모니터링 화면·도구로 확인하고, 장애 시 대응한다. | ACT-S03 |
| UC-12 | 계정·권한 관리 | 플랫폼 로그인 계정·API 키 등 자격 정보의 안전한 저장·폐기·정책 관리(개발/운영 관점). 값 변경은 고객 운영팀이 UC-07로 수행 가능. | — |
