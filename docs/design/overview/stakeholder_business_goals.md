# Stakeholder별 비즈니스 골

## 1. 문서 목적
이 문서는 **Stakeholder(이해관계자)별로 이 프로젝트에서 달성하고자 하는 비즈니스 골**을 정리합니다. 역할별 기대 결과를 명확히 해 목표 정렬과 성공 판단에 활용합니다.

---

## 2. 비즈니스 골 요약

| ID | Stakeholder | 비즈니스 골(한 줄) |
|----|-------------|-------------------|
| SH-01 | 고객 | 등록 시간·재작업 감소, 실무에 바로 쓸 수 있는 안정적인 자동화 도입 |
| SH-02 | 개발 담당 | 요구사항을 안정적으로 구현·운영하고, 변경·확장에 대응 가능한 구조 유지 |
| SH-03 | PM/코디네이터 | 단계별 산출물·승인 시점이 명확히 진행되어 일정·범위가 지켜지도록 조율 |

---

## 3. Stakeholder별 비즈니스 골 (상세)

### 3.1 고객 (SH-01)

| 구분 | 비즈니스 골 | 설명·측정 방향 |
|------|-------------|----------------|
| **시간 단축** | 다수 플랫폼 등록에 걸리는 시간을 줄인다. | 회차·플랫폼당 소요 시간 감소 목표치 설정(예: N% 단축), As-Is 대비 측정 |
| **오류·재작업 감소** | 누락·오입력·검증 실패로 인한 재작업 횟수를 줄인다. | 재작업 건수·원인별 비율 목표, 사전 검증 통과율 등 |
| **실무 적용성** | 일상 업무에 바로 적용 가능한 수준의 품질을 확보한다. | 테스트·시연 수용 기준 충족, 실무자 사용성·리포트 활용도 |
| **범위·리스크 균형** | MVP 범위 내에서 비용·일정·품질 균형을 맞춘다. | MVP 범위 확정, Gate별 승인, 변경 요청 시 영향 범위·승인 절차 |
| **운영 표준화** | 예외·장애 시 대응 절차가 정해져 운영이 예측 가능해진다. | 운영 정책·알림·리포트 형식 합의, 재시도·수동 전환 기준 명시 |

**성공 판단**: KPI(시간·재작업·검증 통과율 등)와 측정 시점을 프로젝트 초기에 합의하고, Gate 4·5에서 달성 여부 확인.

---

### 3.2 개발 담당 (SH-02)

| 구분 | 비즈니스 골 | 설명·측정 방향 |
|------|-------------|----------------|
| **요구사항 구현** | 기능·비기능 요구사항을 명세대로 구현한다. | 요구사항 대비 구현 완료율, 수용 기준 충족 |
| **안정성** | 실패 시 원인·대응이 명확하고, 재시도·복구가 가능한 동작을 보장한다. | 오류 분류·재시도 정책, 장애 시 로그·리포트 품질 |
| **유지보수·확장성** | 플랫폼 추가·화면 변경 시 변경 범위를 최소화할 수 있는 구조를 유지한다. | 플랫폼별 분리·설정화, 코드·설계 문서 유지 |
| **운영 전환** | 시범·정식 운영 전환 시 고객이 결과를 해석하고 조치할 수 있도록 전달·문서를 갖춘다. | 운영 가이드, 점검 체크리스트, 알림·리포트 제공 |

**성공 판단**: 설계·구현 품질, 테스트 통과, 운영 전환 조건 충족, 고객 수용 기준 달성.

---

### 3.3 (선택) 프로젝트 관리/코디네이터 (SH-03)

| 구분 | 비즈니스 골 | 설명·측정 방향 |
|------|-------------|----------------|
| **일정·산출물 조율** | 단계별 산출물과 Gate가 계획대로 진행된다. | 마일스톤·Gate 일정 준수, 산출물 제출·승인 완료 |
| **의사결정 시점 명확화** | 결정이 필요한 항목이 적시에 논의·승인된다. | 정기 리뷰·Gate 회의 진행, 미결 항목·이슈 추적 |
| **변경·이슈 추적** | 범위·일정에 영향을 주는 변경과 이슈가 이력으로 관리된다. | 변경 요청·이슈 로그, 영향 범위·승인 기록 |

**성공 판단**: 단계별 진행률, Gate 통과, 변경·이슈에 대한 추적 가능한 이력 유지.

---

## 4. 프로젝트 수준 비즈니스 골과의 연결

프로젝트 전체에서 먼저 확정하는 4가지( [notion_page_2 §1](../../customer/notion_page_2_execution_flow_and_requirements.md) 비즈니스 골 설정 방법 )와 Stakeholder별 골의 관계:

| 프로젝트 수준 질문 | 주로 반영하는 Stakeholder |
|-------------------|---------------------------|
| 무엇을 얼마나 줄일 것인가 (시간·재작업) | 고객(SH-01) |
| 어떤 품질을 확보할 것인가 (누락·오입력 최소화) | 고객(SH-01), 개발 담당(SH-02) |
| 어떤 범위부터 시작할 것인가 (플랫폼/작품/회차) | 고객(SH-01) |
| 어떤 방식으로 성공을 판단할 것인가 (KPI·측정 시점) | 고객(SH-01), 개발 담당(SH-02), PM/코디(SH-03) |
