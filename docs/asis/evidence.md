아래 템플릿대로 자료를 주시면 Recording Info / Timeline / Pain Points / Candidate Automation Hooks 를 구체적 사실로 채우겠습니다.

### 1. Timestamp 기준 예시

- **녹화 영상 기준**: `MM:SS` (예: 00:00, 01:23, 15:40)
- **실제 시각 기준**: `YYYY-MM-DD HH:MM` 또는 `HH:MM:SS` (예: 2026-02-13 14:30, 14:30:05)
- **상대 시각**: 세션 시작을 00:00으로 두고 `+MM:SS` (예: +00:00, +02:15)
- **선택**: 위 중 하나를 정한 뒤, Timeline 표의 "시각" 컬럼에 일관되게 기입해 주세요.

### 2. 필요한 입력 데이터 샘플 체크리스트

- [ ] 녹화/시연 일시 및 참여자(역할만 기재, 개인정보 제외)
- [ ] 시연에 사용한 도구/화면/URL (민감정보 마스킹)
- [ ] 단계별 타임스탬프가 포함된 메모, 자막, 또는 요약문
- [ ] 각 단계에서 누가(액터) 무엇을 했는지(액션)와 결과물/에러 여부
- [ ] 관찰된 불편/지연/재작업 구간(Pain Points) 목록
- [ ] 자동화 후보가 될 수 있는 반복 구간/트리거 위치

|------|-----------|-------------|
| 1 | Validation Failure | 필수 입력 누락, 형식 오류 시 화면/메시지와 사용자 대응 |
| 2 | Upload Failure | 대용량/네트워크 끊김 시 재시도·에러 메시지·복구 방법 |
| 3 | Duplicate Conflict | 동일 작품/에피소드 재등록 시 안내·중복 방지 여부 |
| 4 | Permission Issue | 권한 없는 메뉴/액션 접근 시 메시지·권한 요청 경로 |
| 5 | Downstream Failure | 외부 API/플랫폼 장애·지연 시 표시·대기·재시도 여부 |
