# 고객 확인 체크리스트 및 회사별 As-Is Flow

Notion에서 관리하던 **고객 확인 체크리스트 (회사별)** 및 **COMPANY-A/B/C As-Is Flow**의 원본입니다.  
회사명(COMPANY-A/B/C)은 실제 업체명(카카오, 네이버, 레진 등)으로 Notion에서 바꿔 사용하시면 됩니다.

| **Step ID** | **단계명** | **현재 방식(요약) – 영상 기준** | **구현 영향 이유(1줄)** | **결정 옵션(선택지)** | **고객 확인** |
| --- | --- | --- | --- | --- | --- |
| ASIS-A-01 | 로그인/세션 | (RIDI) Contents Partners 접속 → ID/PW 로그인. (교보) KYobo 파트너 시스템 접속 → ID/PW 로그인 | 인증 방식이 자동화 난이도를 좌우 | A) ID·비밀번호 / B) SSO / C) API 키 / D) 기타 | [ ] |
| ASIS-A-02 | 워크스페이스·프로젝트 선택 | (RIDI) 상단 드롭다운에서 레이블/계정 선택 후 진행. (교보) 콘텐츠등록 메뉴로 이동 | 고정값이면 설정만으로 처리 가능 | A) 매번 선택 / B) 기본값 고정 / C) 회사별 다름 | [ ] |
| ASIS-A-03 | 작품(Work) 생성/선택 | (RIDI) 신규 작품 등록 또는 회차 등록(기존 작품 선택) 흐름 존재 | 생성/선택에 따라 멱등·ID 매핑이 달라짐 | A) 항상 생성 / B) 기존 선택 위주 / C) 혼합 | [ ] |
| ASIS-A-04 | 에피소드(Item) 메타 입력 | (RIDI) 카테고리/완결/단권·시리즈/제목/소개/목차/나이제한/작가 등 폼에 엑셀 값 복붙. (교보) 콘텐츠등록 폼에 기본정보 입력 | 필수 필드·검증이 입력 스키마에 반영됨 | A) 플랫폼 필수만 / B) 추가 필드 많음 / C) 기타 | [ ] |
| ASIS-A-05 | 원고/파일 업로드 | (추정) 회차별 원고 파일(또는 ZIP/다중 파일) + 필요 시 추가 첨부를 업로드 | 파일 규칙이 패키징·재시도 정책에 영향 | A) 단일 파일 / B) 다중 파일·폴더 / C) 기타 | [ ] |
| ASIS-A-06 | 원고 검증 | (RIDI) 입력 누락 시 경고/가이드 노출(예: 카테고리 필수). 파일 검증 상세는 영상에서 불명확 | 규칙이 업체별이면 validator 분기 필요 | A) 공통 규칙 / B) 전용 규칙 있음 / C) 기타 | [ ] |
| ASIS-A-07 | 에셋 생성·첨부 | (추정) 표지/썸네일 등 에셋 첨부 가능성은 있으나 영상에서 확정적으로 보이지 않음 | 변환/검증 로직 필요 여부에 영향 | A) 제한 있음 / B) 제한 없음 / C) 기타 | [ ] |
| ASIS-A-08 | 최종 검토·수정 | 입력 완료 후 사람이 화면에서 최종 확인 후 저장/등록하는 흐름(제출 클릭은 영상에서 일부 불명확) | 승인 포인트로 둘 경우 워크플로 설계 필요 | A) 자동 통과 / B) 반드시 사람 확인 / C) 기타 | [ ] |
| ASIS-A-09 | 플랫폼 대상 선택·설정 | (교보) 콘텐츠 유형/등록 메뉴로 채널 분기. (RIDI) 도서등록/대량등록 등 메뉴 분기 | 채널·옵션이 설정 스키마에 반영됨 | A) 단일 채널 / B) 다중 채널 / C) 기타 | [ ] |
| ASIS-A-10 | 업로드/발행 요청 | 웹 포털에서 수동 등록(웹 기반). API 사용 흔적 없음 | API 유무가 자동화 방식 결정 | A) API 있음 / B) API 없음(웹만) / C) 기타 | [ ] |
| ASIS-A-11 | 발행 결과 확인·Post-Check | (교보) 공지 팝업 등 UI 인터럽트 존재. 최종 URL/상태 확인은 화면 기반(영상에서 결과 확정 장면은 제한적) | 리포트 스키마·알림 설계에 반영 | A) URL만 / B) URL+상태+에러 / C) 기타 | [ ] |
